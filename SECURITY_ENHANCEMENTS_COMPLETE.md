## 1. ✅ Fixed Security Score Calculation

### Backend Changes (`backend/src/routes/security.ts`)

**Problem:** Score showed 100% even with findings because summary counts weren't updated before calculation.

**Solution:** Added `updateJobSummary(job)` call before score calculation (line 719):

```typescript
// Update summary counts before calculating score
updateJobSummary(job);

// Calculate security score from actual findings
const deductions = (criticalCount * 20) + (highCount * 10) + (mediumCount * 5) + (lowCount * 2);
job.summary.score = Math.max(0, 100 - deductions);
```

**Result:**
- Score now recalculates from actual findings array
- Updates in real-time during audit streaming
- Never shows 100% when findings exist
- Correct deduction formula: Critical=20pts, High=10pts, Medium=5pts, Low=2pts

### Frontend Changes (`frontend/src/pages/Security.tsx`)

**Fixed Summary Cards:**
- Use `auditSummary` counts when available (from backend)
- Fall back to calculated counts only if audit hasn't run
- All cards (Critical, High Priority, Total Findings) now show real counts
- Update immediately when audit completes
